1. Functions are used mostly for calculations, always return something as a result, are called via SELECT, can be embedded in other queries. Procedures, on the other hand, are used to automate complex business logic when it is important to commit transactions in parts, do not necessarily return something, are called via CALL, cannot be called inside SELECT or regular queries. 
2. A trigger cannot be called manually, as it is event-driven. A trigger is tied to a specific table and executes when the event occurs in that table. To artificially test the trigger, t is possible to perform some action on that table.
3. The advantages of using business logic in databases are performance and speed, since then the code is executed next to the data, so operations work faster, and integrity, because when several different services work with the database, the logic inside guarantees that it will be the same for all. The disadvantages are the complexity of debugging, problems with version control and moving to other DBMS that may use a different SQL dialect, which will require rewriting long code.
